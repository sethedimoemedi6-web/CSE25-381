The mechanisms of cyptocurrency:blockchain

This builds on the previous blockchain deep dive but focuses on crptocurrency specific mechanisms

Blockchain is the public ledger that powers crypto.Every transaction is grouped into blocks cryptographically linked and distributed across thousands of nodes

Core Mechanisms

  • Cryptography:Hash functions +digital signatures ensure immutability and ownership
  • Consensus:Nodes agree on the state (PoW,POS)
  • Decentralization:No single point of failure
  • Transparency+pseudonymity:all transactions visible but addresses aren`t tied to real identities (unless exchanged on kyc platforms)

How a crypto transaction actually work

  1. User signs with private key
  2. Broadcast the mempool
  3. Miner/validator includes it in a block
  4. Block is finally confirmed ->finality increases with more blocks

Intermediate Nuance

Blockchains solve the Byzantine Generals Problem:how untrusted parties reach agreement in an adversarial enviroment

Key Takeaway

Blockchain turns trust into code - the backbone that makes cryptocurrency possible without banks

Bitcoin whitepaper and Ethereum white paper

Bitcoin UTXO model

UTXO= Unspent Transaction Output.Bitcoin does not have balances like a bank- it has descrete coins that are spent and created

How UTXOs work

  • Every transaction consumes existing UTXOs as inputs and creates new UTXOs as outputs
  • Your wallet balance =sum of all UTXOs you control
  • Change is sent back to you as new UTXO

Advantages

  • High privacy(harder to link addresses)
  • Parallel validation (Nodes dont need full account history)
  • Prevents double spending natively

Limitations

  • Inefficient for complex apps (hence layer-2)
  • Dust UTXOs can bloat the chain

Intermediate Insight

UTXO is why Bitcoin is extremely secure but less flexible that account based chains

Key Takeaway

UTXO is Bitcoin elegent accounting model flexible that account based chains

Understanding bitcoin and ethereum tokenomics

Bitcoin Tokenomics

  • Fixed Supply:21 million BTC (last coin to be mined~2040)
  • Halvigns every 4 years reduce issuance(next~2028)
  • Deflationary pressure from lost coins +burns
  • Incentive:Miners get block rewards + fees

Ethereum Tokenomics

  • No hard cap but issuance is low (~0.5-1% anually)
  • EIP-1559:Base fee burned (deflationary during high demand)
  • Staking:~30%+ of ETH is staked for security and yield
  • Utility:Gas fees paid in ETH

Comparison

Bitcoin= digital gold (store of value) Ethereum= digital oil (powers the network)

Key Takeaway

Tokenomics align incentives:Bitcoin rewards security through scarcity;Ethereum rewards usage and staking

Ethereum account model

Unlike Bitcoin UTXO Ethereum uses an account-based model(like a bank ledger )

Two account types

  • Externally Owned Accounts(EOAs):Controlled by private keys(your wallet)
  • Contracts Accounts:Controlled by code (smart contracts)

Key Features

  • Nonce:Prevents replay attacks:tracks transaction count
  • Balnce:Direct ETH balance per account
  • Storage root:Contracts store persitent state (variables,mappings)
  • Code hash:immutable smart contracts logic

Advantages over UTXO

  • Easier programming (state changes are straightforward)
  • supports comlex defi and NFTs

Trade-offs

  • Requires careful nonce management
  • State bloat is a challenge (solved via sharding/plans)

Key Takeaway

Ethereum account model it a world computer -programmble and stateful
